# 2026-04-10: Schema Service — custom instance scopes, owner, and type-scoped access

## Overview

The Schema Service now exposes finer-grained access for custom instances, returns ownership metadata on custom instances, and automatically provisions type-specific scopes when a custom entity type is created.

## Granular scopes for custom instances

Custom instance APIs accept **one** of several scopes:

- **Read**: `schema.custominstance_read`, or `{lowerCaseType}.{lowerCaseType}_read`, or `{lowerCaseType}.{lowerCaseType}_readOwn`
- **Manage**: `schema.custominstance_manage`, or `{lowerCaseType}.{lowerCaseType}_manage`, or `{lowerCaseType}.{lowerCaseType}_manageOwn`

Tenant-wide scopes apply to all custom types; type-specific scopes restrict access to a single custom entity type.

## Owner on custom instance responses

Custom instance responses include a read-only **`owner`** object indicating who created the instance: `type` (`EMPLOYEE`, `CUSTOMER`, or `SERVICE`), `userId`, and for customer owners an optional `legalEntityId`.

## Scopes provisioned with new custom entity types

When a new custom schema type is created the platform provisions scopes for that type so clients can read and manage custom instances of that type using the `{lowerCaseType}.{lowerCaseType}_*` patterns above.
