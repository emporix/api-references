# Emporix Documentation Style Guide

This style guide defines how to plan, write, and maintain documentation in the Emporix Documentation Portal. It helps technical writers and engineers create consistent, actionable, and accurate content for business and technical audiences.

## Ground rules

* User personas – Documentation serves both business-oriented decision makers and technical users such as implementation engineers and developers. API content primarily serves software engineers.
* Voice and tone – Use a helpful, user-oriented tone. Prefer direct and concise language over formal or abstract language.
* Golden rule – Clarity over cleverness.
* Practicality first – Each page should help users complete a task, understand a concept, or make a decision.
* Accuracy over speed – If behavior is not confirmed, mark assumptions and validate before publishing.

## How to use this guide

* Start with writing standards for language, structure, and terminology.
* Apply visual standards for diagrams, images, and videos.
* Use API tutorials and API references standards when writing how-tos or OpenAPI specs in `api-references`.
* Use templates for repeatable document types such as changelogs, release notes, and API references.
* When in doubt, optimize for:
  * fast user comprehension
  * actionable examples
  * predictable page structure
